Problems Claude Code Users Face

CCB solves common frustrations with intelligent memory and specialized expertise

Problem: Context Loss

Claude forgets previous conversations. You have to repeat your project's architecture, coding standards, and decisions every single session.

Solution: Project Memory

Knowledge Graph remembers your architecture decisions, coding patterns, and past solutions permanently across sessions.

Problem: Generic Responses

Claude gives generic answers without understanding your specific project context, tech stack, or coding conventions.

Solution: Smart Routing

Smart routing classifies the task type and applies the right capability profile for context-aware responses.

Problem: Expensive Token Costs

All tasks use expensive models even for simple work, wasting tokens and budget on routine operations.

Solution: Smart Model Selection

Routes tasks by complexity: Haiku (simple, cheap) → Sonnet (balanced) → Opus (complex, quality). Saves ~40% token costs.

Problem: Repetitive Context

Explaining your project structure and decisions every conversation wastes time and breaks flow.

Solution: Project Memory

Record decisions and changes in the local knowledge graph → Recall with buddy-remember → No repeated explanations needed.

Designed to Evolve with Your Project

Instead of treating each prompt as a clean slate, CCB maintains a persistent understanding of your codebase, architectural decisions, and workflow patterns.

As you use it, the plugin progressively aligns with how your project is structured and how you prefer to work—reducing repetition, improving consistency, and making AI assistance feel less like a tool and more like a long-term collaborator.

How It Works: Behavioral Evolution

CCB does not retrain models. Instead, it uses a persistent system layer inside the Plugin:

Project Memory Accumulation

Architectural decisions, conventions, and resolved discussions are stored as structured context.

Pattern Reinforcement

Repeated approvals, refactors, and preferred solutions gradually shape how future prompts are constructed.

Task-Type Differentiation

Review, debugging, refactoring, and design tasks are handled with different internal prompting strategies.

Context Reuse

Relevant memory is selectively injected, avoiding prompt bloat while preserving continuity.

Quick Start

Get up and running in 2 minutes

1

Prerequisites

Node.js 20+ and Claude Code installed 

# Check Node.js version
node --version  # Should be v18 or higher
2

Clone & Install 

# Clone the repository
git clone https://github.com/PCIRCLE-AI/claude-code-buddy.git
cd claude-code-buddy

# Run interactive installer
./scripts/install.sh
3

Start Using

Restart Claude Code and try natural language commands:

"Analyze my codebase"
"Review for security"

No API keys needed for core features - CCB uses your existing Claude Code subscription.

How to Use CCB

Two ways to use: Automatic (just talk normally) or Buddy Commands (explicit control)

Automatic Mode

Just use Claude Code normally. CCB enhances your requests in the background: 

You: "Review this code for security"
     ↓
CCB: Detects → code review task
CCB: Applies → review guidance profile
CCB: Enhances → security checklist
     ↓
Claude: Expert security review

Buddy Commands

Use explicit commands for specific features:

buddy-do "setup authentication"

Execute with smart routing

buddy-remember "API decisions"

Search project memory

get-session-health

Check session health

Project Memory

buddy-remember "API decisions" Recall design choices
buddy-remember "test failures" Track recent issues
buddy-remember "naming conventions" Stay consistent

Smart Planning

Break down features into small, ordered steps: 

generate-smart-plan {"featureDescription":"OAuth login"}

Includes tasks, dependencies, and testing guidance.

Session Health

Monitor context quality and token usage:

get-session-health Check token usage and warnings
buddy-remember "auth decisions" Recall project memory
generate-smart-plan Break down the next feature

# Git Assistant UX Design（設計稿 - 待審查）

> **狀態**: 🔍 等待用戶審查和批准
> **目的**: 設計友善的 Git 助手，自動幫助用戶管理版本控制
> **原則**: 教育優先、徵求許可、漸進自動化

---

## 🎯 設計目標

1. **對新手友善** - 用日常語言解釋 Git，不用技術術語
2. **徵求許可** - 永遠先問再做，尊重用戶選擇
3. **漸進自動化** - 從手動 → 半自動 → 全自動
4. **記住偏好** - 學習用戶習慣，減少重複提問
5. **可撤銷** - 所有操作都可以回退

---

## 📊 兩大使用場景

### 場景 A: 專案沒有 Git（新專案 or 現有專案）
### 場景 B: 專案已有 Git（持續維護）

# 🎨 場景 B: 專案已有 Git（持續維護）

## 觸發時機

```
觸發條件：
1. 檔案修改累積到一定程度（例如 10 個檔案或 100 行修改）
2. 時間間隔（例如每 30 分鐘）
3. 用戶完成一個功能（推測：新增/修改/刪除檔案模式）
4. 用戶主動儲存重要檔案
5. 用戶準備關閉編輯器

智能判斷：
- 小修改：不提示
- 中等修改：溫和提醒
- 大修改：主動建議
```

## 自動化等級設計

### Level 0: 完全手動（預設給新手）

```
用戶完全控制，Smart-agents 只提醒
```

### Level 1: 智能提醒（推薦）

```
Smart-agents 提醒 + 建議，用戶確認
```

### Level 2: 半自動（進階）

```
Smart-agents 自動準備，用戶快速批准
```

### Level 3: 全自動（專家，需明確啟用）

```
Smart-agents 全自動執行，事後通知
```

## 智能時機判斷

### 何時提醒儲存版本？

```typescript
interface SaveTrigger {
  // 檔案數量門檻
  filesChanged: number;     // 例如：10 個檔案

  // 修改行數門檻
  linesChanged: number;     // 例如：100 行

  // 時間間隔
  timeSinceLastSave: number; // 例如：30 分鐘

  // 工作模式偵測
  workPattern: {
    created: string[];      // 新增的檔案
    modified: string[];     // 修改的檔案
    deleted: string[];      // 刪除的檔案
  };

  // 智能判斷
  aiConfidence: number;     // AI 判斷「適合儲存」的信心分數
}

// 觸發條件範例
const shouldSuggestSave =
  (filesChanged >= 10) ||
  (linesChanged >= 100) ||
  (timeSinceLastSave >= 30 * 60 * 1000) ||
  (aiConfidence > 0.8);  // AI 高度確信適合儲存
```

### AI 工作模式偵測

```typescript
// 範例：偵測「完成一個功能」
Pattern: 新功能開發
  ✅ 新增組件檔案 (*.tsx, *.vue)
  ✅ 新增 API 檔案 (api/*.ts)
  ✅ 更新路由 (routes/*.ts)
  ✅ 更新依賴 (package.json)
  → 信心分數：95% → 建議儲存

Pattern: Bug 修復
  ✅ 修改少量檔案 (1-3 個)
  ✅ 小幅度修改 (<50 行)
  ✅ 包含測試檔案
  → 信心分數：85% → 建議儲存

Pattern: 探索/實驗
  ⚠️ 頻繁修改同一檔案
  ⚠️ 多次復原
  ⚠️ 不穩定的修改模式
  → 信心分數：30% → 不建議儲存（等更穩定）
```

   - 經常跳過通知 → 降低通知頻率

   - 用戶習慣的描述格式
   - 常用的關鍵字

   - 通常什麼時候想儲存版本
   - 偏好的提醒時機

4. 反饋學習
   - 用戶接受 AI 建議 → 提高類似情況的信心分數
   - 用戶拒絕 AI 建議 → 降低類似情況的信心分數

# 🔧 技術實作鉤子（Hook Points）

## Hook 觸發點設計

```typescript
// 1. 專案初始化/開啟
Hook: 'project:init'
Hook: 'project:open'
→ 檢查是否有 .git
→ 如果沒有，觸發「場景 A: Git 設定助手」

// 2. 檔案變更監控
Hook: 'file:changed'
Hook: 'file:saved'
→ 累積變更統計
→ 達到門檻時，觸發「場景 B: 儲存版本提醒」

// 3. 工作流程事件
Hook: 'workflow:feature-complete'  // AI 判斷功能完成
Hook: 'workflow:bug-fixed'         // AI 判斷 bug 修復
→ 智能建議儲存版本

// 4. 用戶操作
Hook: 'user:closing-editor'        // 用戶準備關閉
Hook: 'user:switching-branch'      // 用戶切換分支
→ 提醒未儲存的變更

// 5. 定時檢查
Hook: 'timer:interval'             // 每 N 分鐘檢查
→ 評估是否需要提醒
```
